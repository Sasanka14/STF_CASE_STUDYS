Framed Question for Case Study

**Case Study 96: File Backup Creation (Advanced)**

A student wants to ensure that their important notes are always backed up automatically. To achieve this, a program must be developed in C that creates a duplicate copy of a given file with a different name or extension. The program should use **system calls (`open`, `read`, `write`, `close`)** to ensure redundancy at the operating system level.

The solution should also demonstrate **error handling, file size reporting, and progress tracking** while copying, thus reinforcing the concepts of sequential file access and system-level programming.

#### Requirements:

1. Input file name and backup file name must be provided as **command-line arguments**.
2. The program must:

   * Open the source file in read-only mode.
   * Create/overwrite the backup file in write mode.
   * Copy the file in chunks using a buffer.
   * Show progress of copying in percentage.
   * Report the size of the original file after completion.
   * Close both files safely with error checks.

---

#### Deliverables:

* **Source Code**: `backup.c`
* **Input File** (e.g., `notes.txt`)
* **Backup File** (e.g., `notes_backup.txt`)
* **Execution Screenshot** showing file backup with progress and file size report.
